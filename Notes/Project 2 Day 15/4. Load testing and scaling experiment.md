അടിപൊളി ചേട്ടായി! **`cpu: 0%/50%`** എന്ന് വന്നിട്ടുണ്ടെങ്കിൽ അതിന്റെ അർത്ഥം Metrics Server കൃത്യമായി ഡാറ്റ ശേഖരിക്കുന്നുണ്ട് എന്നാണ്. ഇപ്പോൾ നമ്മുടെ ആപ്പിന് പണിയൊന്നുമില്ലാത്തതുകൊണ്ട് അത് ശാന്തമായി ഇരിക്കുകയാണ്.

ഇനി നമുക്ക് ഇതിനെയൊന്ന് പരീക്ഷിക്കാം! ലോഡ് കൂടുമ്പോൾ എങ്ങനെയാണ് കുബെർനെറ്റിസ് പുതിയ പോഡുകൾ ഉണ്ടാക്കുന്നത് എന്ന് ന്റെ ചേട്ടായിക്ക് നേരിട്ട് കാണാം.

---

### **Step 4: Load Testing (കൃത്രിമമായി ലോഡ് ഉണ്ടാക്കാം)**

നമുക്ക് ഒരു താൽക്കാലിക പോഡ് ഉണ്ടാക്കി അതിൽ നിന്ന് നിരന്തരം നമ്മുടെ API-ലേക്ക് റിക്വസ്റ്റുകൾ വിടാം. ഇതിനായി താഴെ പറയുന്ന കമാൻഡ് കോപ്പി ചെയ്ത് ടെർമിനലിൽ അടിക്കൂ:

```bash
kubectl run -i --tty load-generator --rm --image=busybox:1.28 --restart=Never -- /bin/sh -c "while sleep 0.01; do wget -q -O- http://sentiment-api; done"
```

> **കുറിപ്പ്:** ഈ കമാൻഡ് അടിക്കുമ്പോൾ സ്ക്രീൻ ലോഡ് ആയിക്കൊണ്ടിരിക്കും. അത് അവിടെ തന്നെ ഇരിക്കട്ടെ.

---

### **Step 5: സ്കെയിലിംഗ് നിരീക്ഷിക്കാം**

ഇനി ന്റെ ചേട്ടായി **പുതിയൊരു ടെർമിനൽ വിൻഡോ** തുറക്കുക (അല്ലെങ്കിൽ നിലവിലുള്ളത് ഡ്യൂപ്ലിക്കേറ്റ് ചെയ്യുക). എന്നിട്ട് താഴെ പറയുന്ന കമാൻഡ് അടിക്കുക:

```bash
kubectl get hpa sentiment-api-hpa --watch
```



അല്പസമയം കഴിയുമ്പോൾ ചേട്ടായിക്ക് താഴെ പറയുന്ന മാറ്റങ്ങൾ കാണാം:
1.  **TARGETS:** `0%/50%` എന്നുള്ളത് പതുക്കെ `100%/50%` അല്ലെങ്കിൽ അതിനു മുകളിലേക്ക് പോകും.
2.  **REPLICAS:** `2` എന്നുള്ളത് മാറി `3`, `4`, ഒടുവിൽ `5` എന്നാകും.

അത് ശരിയാണോ എന്ന് ഉറപ്പിക്കാൻ `kubectl get pods` അടിച്ചു നോക്കിയാൽ 5 പോഡുകൾ ഓടുന്നത് കാണാം!

---

### **പ്രശ്നങ്ങൾ ഉണ്ടാകാൻ സാധ്യതയുള്ള ഇടങ്ങൾ (Debugging Tips):**

ഒരുവേള `TARGETS` എന്നതിൽ `<unknown>` എന്ന് തന്നെ തുടരുകയാണെങ്കിൽ:
* **Metrics Server Issues:** നേരത്തെ നമ്മൾ ഇൻസ്റ്റാൾ ചെയ്ത മെട്രിക്സ് സർവർ റെഡിയാണോ എന്ന് `kubectl get pods -n kube-system` അടിച്ചു നോക്കണം.
* **Resource Limits:** `deployment.yaml`-ൽ `cpu requests` നൽകിയിട്ടുണ്ടോ എന്ന് ഉറപ്പാക്കണം (നമ്മൾ അത് നേരത്തെ നൽകിയിട്ടുണ്ട്, അതുകൊണ്ട് കുഴപ്പമില്ല).



**ന്റെ ചേട്ടായി അത് പരീക്ഷിച്ചു നോക്കിയോ?** പോഡുകൾ കൂടുന്നത് കാണുമ്പോൾ ഒരു പ്രത്യേക സന്തോഷമാണ്! ലോഡ് ടെസ്റ്റ് നിർത്തിയ ശേഷം കുറച്ചു സമയം കഴിഞ്ഞാൽ പോഡുകൾ തനിയെ 2 എണ്ണമായി കുറയുകയും ചെയ്യും.

**ഇത് കഴിഞ്ഞാൽ നമുക്ക് അടുത്ത ഫേസ് ആയ Load Balancing (ALB/Ingress) ലേക്ക് പോകാം. റെഡിയാണോ ന്റെ ചേട്ടായി?**
