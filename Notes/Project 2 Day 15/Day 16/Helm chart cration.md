
നമ്മൾ ഇതുവരെ ചെയ്തത്:
1. ഒരു API ഉണ്ടാക്കി Dockerize ചെയ്തു.
2. അത് Kubernetes-ൽ നേരിട്ട് (YAML ഫയലുകൾ വഴി) ഡിപ്ലോയ് ചെയ്തു.
3. Prometheus & Grafana സെറ്റപ്പ് ചെയ്തു.

ഇനി നമ്മൾ ചെയ്യാൻ പോകുന്നത് **Step 2: Helm Charts (Repackaging our API)** ആണ്. 

---

### **എന്തിനാണ് നമ്മൾ ഹെൽം ചാർട്ട് ഉണ്ടാക്കുന്നത്?**

ഇതുവരെ നമ്മൾ `deployment.yaml`, `service.yaml`, `ingress.yaml` എന്നിങ്ങനെ ഓരോ ഫയലുകളും പ്രത്യേകം പ്രത്യേകം `kubectl apply` ചെയ്യുകയായിരുന്നു. എന്നാൽ ഒരു പ്രൊഫഷണൽ എൻവയോൺമെന്റിൽ ഇത് ശരിയല്ല. 
* ഒരു മാറ്റം വരുത്തണമെങ്കിൽ എല്ലാ ഫയലുകളിലും പോയി എഡിറ്റ് ചെയ്യണം. 
* പഴയ വേർഷനിലേക്ക് തിരിച്ചു പോകാൻ (Rollback) ബുദ്ധിമുട്ടാണ്.

ഹെൽം ചാർട്ട് ഉണ്ടാക്കിയാൽ ന്റെ ചേട്ടായിക്ക് ഈ ഫയലുകളെല്ലാം ഒരൊറ്റ പാക്കേജ് ആക്കി മാറ്റാം. മാറ്റങ്ങൾ വരുത്താൻ `values.yaml` മാത്രം നോക്കിയാൽ മതി.

---

### **നമുക്ക് തുടങ്ങാം ചേട്ടായി!**

ആദ്യം നമ്മുടെ API-ക്ക് വേണ്ടിയുള്ള ഹെൽം ചാർട്ട് സ്ട്രക്ചർ ഉണ്ടാക്കാം.

**1. പ്രൊജക്റ്റ് ഫോൾഡറിലേക്ക് പോകുക:**
```bash
cd ~/sentiments-api-project
```

**2. പുതിയൊരു ഹെൽം ചാർട്ട് ക്രിയേറ്റ് ചെയ്യുക:**
നമുക്ക് ഇതിന് `sentiment-chart` എന്ന് പേരിടാം.
```bash
helm create sentiment-chart
```

**3. ഇപ്പോൾ ഉണ്ടായ ഫയലുകൾ ഒന്ന് നോക്കൂ:**
```bash
ls -R sentiment-chart
```



---

### **ഇനി നമ്മൾ ചെയ്യേണ്ടത്:**

ഈ `sentiment-chart/templates` ഫോൾഡറിനുള്ളിൽ കുറെ ഡിഫോൾട്ട് ഫയലുകൾ ഉണ്ടാകും. നമുക്ക് അവ മാറ്റി നമ്മൾ നേരത്തെ ഉണ്ടാക്കിയ `k8s/*.yaml` ഫയലുകളെ ഇതിലേക്ക് ഇന്റഗ്രേറ്റ് ചെയ്യണം. 

**അടുത്തതായി ന്റെ ചേട്ടായി ചെയ്യേണ്ടത്:**
ഈ `templates` ഫോൾഡറിനുള്ളിലെ ആവശ്യമില്ലാത്ത ഫയലുകൾ ഒന്ന് ക്ലീൻ ചെയ്യാം. താഴെ പറയുന്ന കമാൻഡ് അടിക്കൂ:

```bash
rm -rf sentiment-chart/templates/*
```


-------------------
നമുക്ക് നമ്മുടെ `k8s` ഫോൾഡറിലെ ഫയലുകളെ ഒരു പ്രൊഫഷണൽ ഹെൽം ചാർട്ട് ആക്കി മാറ്റാം. 

ഹെൽം ചാർട്ടിന്റെ പവർ എന്താണെന്നുവെച്ചാൽ, നമുക്ക് YAML ഫയലുകളിലെ ഡാറ്റ നേരിട്ട് എഴുതാതെ, പകരം **Variables** (ചേട്ടായിക്ക് വേണമെങ്കിൽ മാറ്റാവുന്ന വിലകൾ) ഉപയോഗിക്കാം.

---

### **Step 1: ഫയലുകൾ Templates-ലേക്ക് മാറ്റാം**

ആദ്യം ന്റെ ചേട്ടായിയുടെ `k8s` ഫോൾഡറിലെ ഫയലുകൾ (IAM Policy ഒഴികെ) ഹെൽം ചാർട്ടിന്റെ ടെംപ്ലേറ്റുകളിലേക്ക് കോപ്പി ചെയ്യാം.

ഈ കമാൻഡ് അടിക്കൂ:
```bash
cp ~/sentiments-api-project/k8s/*.yaml ~/sentiments-api-project/sentiment-chart/templates/
```

ഇപ്പോൾ `sentiment-chart/templates/` ഫോൾഡറിൽ നമ്മുടെ പഴയ ഫയലുകൾ എല്ലാം വന്നിട്ടുണ്ടാകും.

---

### **Step 2: Variables (Values.yaml) സെറ്റ് ചെയ്യാം**

ഇനി നമ്മൾ ചെയ്യാൻ പോകുന്നത് ഈ ഫയലുകളെ **Dynamic** ആക്കുക എന്നതാണ്. ഉദാഹരണത്തിന്, നാളെ ന്റെ ചേട്ടായിക്ക് ഇമേജ് വേർഷൻ മാറ്റണമെങ്കിൽ ഓരോ ഫയലിലും പോകേണ്ടതില്ല, പകരം `values.yaml`-ൽ മാത്രം മാറ്റിയാൽ മതി.

ആദ്യം `sentiment-chart/values.yaml` എന്ന ഫയൽ തുറന്ന് അതിലുള്ള പഴയ കാര്യങ്ങൾ എല്ലാം കളഞ്ഞ് താഴെ പറയുന്നവ പേസ്റ്റ് ചെയ്യൂ:
```bash
vim ~/sentiments-api-project/sentiment-chart/values.yaml
```

**താഴെ കാണുന്നതാണ് നമ്മുടെ പുതിയ Values:**
```yaml
replicaCount: 2

image:
  repository: <ന്റെ ചേട്ടായിയുടെ-docker-hub-id>/sentiment-api
  tag: latest
  pullPolicy: Always

service:
  type: ClusterIP
  port: 80

ingress:
  groupName: "sentiments-group"
  scheme: internet-facing
  targetType: ip

hpa:
  minReplicas: 2
  maxReplicas: 5
  cpuTarget: 50
```

---

### **Step 3: ഒരു ഫയൽ ഹെൽം ടെംപ്ലേറ്റ് ആക്കി മാറ്റാം (ഉദാഹരണം)**

ഇനി നമുക്ക് `deployment.yaml` ഒന്ന് എഡിറ്റ് ചെയ്യാം. അതിൽ ഇമേജിന്റെ പേര് നേരിട്ട് കൊടുക്കുന്നതിന് പകരം ഹെൽം വേരിയബിൾ കൊടുക്കണം.

`vim ~/sentiments-api-project/sentiment-chart/templates/deployment.yaml` തുറന്ന് ഇമേജ് വരുന്ന ഭാഗം ഇങ്ങനെ മാറ്റൂ:



```yaml
# deployment.yaml ഉള്ളിൽ ഇങ്ങനെ മാറ്റുക
    spec:
      containers:
      - name: sentiment-api
        image: {{ .Values.image.repository }}:{{ .Values.image.tag }} # ഇതാണ് ഹെൽം മാജിക്!
        ports:
        - containerPort: 8000
```

---

### **അടുത്ത നീക്കം:**

ന്റെ ചേട്ടായി എല്ലാ ഫയലുകളും ഇപ്പോൾ തന്നെ മാറ്റാൻ നിൽക്കണ്ട. ആദ്യം ഈ ഫയലുകൾ കോപ്പി ചെയ്തിട്ട് `values.yaml` അപ്‌ഡേറ്റ് ചെയ്യൂ. 

**ഒരു കാര്യം ചോദിക്കട്ടെ ചേട്ടായി:** ന്റെ ചേട്ടായിയുടെ Docker Hub ID എന്തായിരുന്നു? അത് വെച്ച് വേണം നമുക്ക് `values.yaml` സെറ്റ് ചെയ്യാൻ. 

അതുപോലെ, നമ്മുടെ പഴയ `kubectl apply` വഴി ഡിപ്ലോയ് ചെയ്ത സാധനങ്ങൾ നമുക്ക് ഒന്ന് ക്ലീൻ ചെയ്യണ്ടേ? ഹെൽം വഴി പുതിയത് ഇൻസ്റ്റാൾ ചെയ്യുന്നതിന് മുൻപ് അത് ചെയ്യുന്നതാണ് നല്ലത്.

**ന്റെ ചേട്ടായി ഫയലുകൾ കോപ്പി ചെയ്തു കഴിഞ്ഞോ?** കഴിഞ്ഞെങ്കിൽ നമുക്ക് ബാക്കി ഫയലുകളും (Ingress, HPA) വേരിയബിൾസ് വെച്ച് ലിങ്ക് ചെയ്യാം!
