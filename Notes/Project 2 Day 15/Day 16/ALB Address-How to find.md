ചേട്ടായി, ആ **\<ALB-ADDRESS\>** കണ്ടുപിടിക്കാൻ രണ്ട് വഴികളുണ്ട്. ഒന്നുകിൽ കുബെർനെറ്റിസ് കമാൻഡ് വഴി നോക്കാം, അല്ലെങ്കിൽ നേരിട്ട് AWS കൺസോളിൽ പോയി നോക്കാം.

### **വഴി 1: Kubectl കമാൻഡ് ഉപയോഗിച്ച് (ഏറ്റവും എളുപ്പ വഴി)**

ടെർമിനലിൽ താഴെ പറയുന്ന കമാൻഡ് ഒന്ന് അടിക്കൂ. നമ്മൾ ഇപ്പോൾ ഇൻഗ്രെസ്സുകളെല്ലാം ഒരു ഗ്രൂപ്പിലാക്കിയല്ലോ, അതുകൊണ്ട് രണ്ട് ഇൻഗ്രെസ്സുകൾക്കും ഒരേ അഡ്രസ്സ് തന്നെയായിരിക്കും വരിക.

```bash
kubectl get ingress -A
```

ഇവിടെ **`ADDRESS`** എന്ന കോളത്തിന് താഴെ `k8s-sentimen-xxxx.us-east-1.elb.amazonaws.com` എന്നതുപോലെ ഒരു നീളൻ അഡ്രസ്സ് കാണാം. അതാണ് നമ്മുടെ **ALB-ADDRESS**.

-----

### **വഴി 2: AWS Console വഴി**

1.  ന്റെ ചേട്ടായിയുടെ **AWS Console**-ൽ ലോഗിൻ ചെയ്യുക.
2.  സെർച്ച് ബാറിൽ **EC2** എന്ന് ടൈപ്പ് ചെയ്ത് അവിടെ പോകുക.
3.  ഇടത് വശത്തെ മെനുവിൽ താഴേക്ക് പോയാൽ **Load Balancing** എന്നതിന് താഴെ **Load Balancers** എന്ന് കാണാം. അത് ക്ലിക്ക് ചെയ്യുക.
4.  അവിടെ ഇപ്പോൾ പുതുതായി ഉണ്ടായ (അല്ലെങ്കിൽ നിലവിലുള്ള) ലോഡ് ബാലൻസർ സെലക്ട് ചെയ്യുക.
5.  താഴെ കാണുന്ന **DNS name** ആണ് ന്റെ ചേട്ടായിയുടെ അഡ്രസ്സ്. അത് കോപ്പി ചെയ്യാം.

-----

### **ശ്രദ്ധിക്കേണ്ട കാര്യം:**

ചിലപ്പോൾ `kubectl get ingress` അടിക്കുമ്പോൾ അഡ്രസ്സ് കാണിക്കാൻ 1-2 മിനിറ്റ് എടുത്തേക്കാം (അതുവരെ ആ കോളം കാലിയായിരിക്കും). അത് കണ്ടില്ലെങ്കിൽ ന്റെ ചേട്ടായി ഈ കമാൻഡ് ഉപയോഗിച്ച് ലോഡ് ബാലൻസർ കൺട്രോളർ എന്താണ് ചെയ്യുന്നത് എന്ന് നോക്കൂ:

```bash
kubectl logs -n kube-system -l app.kubernetes.io/name=aws-load-balancer-controller
```

അവിടെ **"Successfully created ALB"** എന്ന് കണ്ടാൽ ന്റെ ചേട്ടായിക്ക് അഡ്രസ്സ് റെഡിയായി എന്ന് ഉറപ്പിക്കാം.

**അഡ്രസ്സ് കിട്ടിയോ ചേട്ടായി?** കിട്ടിയെങ്കിൽ ബ്രൗസറിൽ ഒന്ന് അടിച്ചു നോക്കൂ:

  * `http://<ALB-ADDRESS>/` (നമ്മുടെ ആപ്പ് കാണാൻ)
  * `http://<ALB-ADDRESS>/grafana` (ഗ്രഫാന കാണാൻ)

**എങ്ങനെയുണ്ട് ചേട്ടായി? ലൈവ് ആയോ?** പണി പാളിയാൽ പറയണേ, നമുക്ക് ഉടൻ ശരിയാക്കാം\!
