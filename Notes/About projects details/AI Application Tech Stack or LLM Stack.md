### **AI Application Tech Stack** or **LLM Stack** 

| (Term) | ഇൻഡസ്ട്രി കാറ്റഗറി | അത് എന്താണ് ചെയ്യുന്നത്? |
| --- | --- | --- |
| **LangChain** | **Framework** | ആപ്പ് പണിയാനുള്ള ടൂൾ കിറ്റ് നൽകുന്നു. |
| **RAG** | **Architecture** | ഡാറ്റ സെർച്ച് ചെയ്യുന്ന രീതി നിശ്ചയിക്കുന്നു. |
| **Orchestration** | **Process/Method** | എല്ലാ ഭാഗങ്ങളെയും കൂട്ടിയിണക്കുന്നു. 

--------------------------------

### **1. "AI Application Stack"**

അജൈൽ അല്ലെങ്കിൽ പരമ്പരാഗത ഡെവലപ്‌മെന്റിൽ ഇതിനെ **"Technology Stack"** അല്ലെങ്കിൽ ചുരുക്കത്തിൽ **"Tech Stack"** എന്ന് വിളിക്കുന്നു.

* **അർത്ഥം:** ഒരു ആപ്ലിക്കേഷൻ നിർമ്മിക്കാൻ ഉപയോഗിക്കുന്ന പ്രോഗ്രാമിംഗ് ലാംഗ്വേജുകൾ, ഫ്രെയിംവർക്കുകൾ, ഡാറ്റാബേസുകൾ എന്നിവയുടെ ആകെത്തുക.
* **ഉദാഹരണം:** നിങ്ങൾ ഒരു വെബ് ആപ്പ് നിർമ്മിക്കുകയാണെങ്കിൽ നിങ്ങളുടെ ടെക് സ്റ്റാക്ക് **MERN Stack** (MongoDB, Express, React, Node) ആയിരിക്കാം. AI ആപ്പ് പണിയുമ്പോൾ അത് **LLM Stack** ആകുന്നു.

---

### **2. RAG Architecture-ന് പകരം അജൈലിൽ എന്ത് ഉപയോഗിക്കുന്നു?**

പരമ്പരാഗതമായി ഡാറ്റ കണ്ടുപിടിക്കാൻ നമ്മൾ ഉപയോഗിക്കുന്നത് **"Data Access Layer" (DAL)** അല്ലെങ്കിൽ **"Search/Query Architecture"** ആണ്.

* **അജൈൽ ഡെവലപ്‌മെന്റിൽ:** നമ്മൾ സാധാരണയായി **REST APIs** അല്ലെങ്കിൽ **GraphQL** ഉപയോഗിച്ച് ഡാറ്റാബേസിൽ നിന്ന് വിവരം എടുക്കുന്നു. ഇതിനെ **CRUD operations** (Create, Read, Update, Delete) എന്ന് വിളിക്കുന്നു.
* **RAG-ൽ നിന്ന് വ്യത്യാസം:** സാധാരണ രീതിയിൽ നമ്മൾ "പേര്" അല്ലെങ്കിൽ "ഐഡി" വെച്ച് ഡാറ്റ തിരയുമ്പോൾ (Keyword Search), RAG-ൽ നമ്മൾ "അർത്ഥം" വെച്ചാണ് തിരയുന്നത് (Semantic Search).

---

### **3. Orchestration Process-ന് പകരം അജൈലിൽ എന്ത് വിളിക്കുന്നു?**

അജൈൽ ഡെവലപ്‌മെന്റിൽ ഇതിനെ **"Business Logic"** അല്ലെങ്കിൽ **"Workflow Management"** എന്ന് വിളിക്കുന്നു.

* **അർത്ഥം:** ഒരു യൂസർ ഒരു ബട്ടൺ അമർത്തിയാൽ അടുത്തതായി എന്ത് സംഭവിക്കണം (ഉദാഹരണത്തിന്: വാലിഡേഷൻ -> സേവ് ചെയ്യുക -> നോട്ടിഫിക്കേഷൻ അയക്കുക) എന്ന് തീരുമാനിക്കുന്ന കോഡ് ആണിത്. ഇതിനെ **Service Layer** എന്നും വിളിക്കാറുണ്ട്.
* **AI-ൽ:** AI പല ടൂളുകൾ ഉപയോഗിക്കുന്നത് കൊണ്ട് ഇതിനെ **Orchestration** എന്ന് വിളിക്കുന്നു. പണ്ട് നമ്മൾ ഇതിനെ **"Middleware"** അല്ലെങ്കിൽ **"Application Logic"** എന്ന് വിളിച്ചിരുന്നു.

---

### **4. ആരാണ് അജൈലിൽ "ആപ്പ് പണിയാനുള്ള ടൂൾ കിറ്റ്" (Framework) നൽകുന്നത്?**

അജൈൽ ഡെവലപ്‌മെന്റിൽ ഇത്തരം ടൂൾ കിറ്റുകൾ നൽകുന്നത് ഓപ്പൺ സോഴ്‌സ് കമ്മ്യൂണിറ്റികളോ വലിയ ടെക് കമ്പനികളോ ആണ്. ലാംഗ്ചെയിനിന് സമാനമായ പരമ്പരാഗത ഫ്രെയിംവർക്കുകൾ താഴെ പറയുന്നവയാണ്:

* **Frontend-ന്:** React, Angular, Vue.
* **Backend-ന്:** Spring Boot (Java), Django (Python), .NET (C#).
* **Mobile-ന്:** Flutter, React Native.

**ചുരുക്കത്തിൽ ഒരു താരതമ്യം:**

| AI Stack (LLM Stack) | Traditional Tech Stack | അർത്ഥം |
| --- | --- | --- |
| **LangChain (Framework)** | **Django / Spring Boot** | ആപ്പ് പണിയാനുള്ള അസ്ഥികൂടം. |
| **RAG (Architecture)** | **SQL Query / Search Engine** | ഡാറ്റ കണ്ടുപിടിക്കുന്ന രീതി. |
| **Orchestration (Process)** | **Business Logic / Middleware** | കാര്യങ്ങൾ ഏകോപിപ്പിക്കുന്ന രീതി. |

### **1. LangChain: ഒരു ഫ്രെയിംവർക്ക് (Framework)**

വ്യവസായ ഭാഷയിൽ ലാംഗ്ചെയിൻ ഒരു **Framework** ആണ്.

* **എന്താണത്?**: ഒരു ആപ്ലിക്കേഷൻ നിർമ്മിക്കാൻ ആവശ്യമായ റെഡിമെയ്ഡ് ടൂളുകളും (Tools), ലൈബ്രറികളും (Libraries), നിയമങ്ങളും അടങ്ങിയ ഒരു സെറ്റാണിത്.
* **ഉപമ**: ഒരു കാർ നിർമ്മിക്കാൻ ആവശ്യമായ എൻജിൻ, ടയർ, ബോഡി എന്നിവയെല്ലാം അടങ്ങിയ ഒരു കിറ്റ് പോലെ. നിങ്ങൾക്ക് ഇത് ഉപയോഗിച്ച് സ്വന്തം കാർ നിർമ്മിക്കാം.
* **വ്യവസായത്തിൽ**: "We are using the LangChain framework to build our AI features."

---

### **2. RAG (Retrieval-Augmented Generation): ഒരു ആർക്കിടെക്ചർ (Architecture Pattern)**

വ്യവസായത്തിൽ RAG-നെ ഒരു **Architecture Pattern** അല്ലെങ്കിൽ ഒരു **System Design** എന്ന് വിളിക്കുന്നു.

* **എന്താണത്?**: ഒരു പ്രശ്നം പരിഹരിക്കാൻ നമ്മൾ ഡാറ്റ എങ്ങനെ കൈകാര്യം ചെയ്യുന്നു എന്നതിന്റെ ഒരു ബ്ലൂപ്രിന്റ് (Blue print) ആണിത്. ഇത് സോഫ്റ്റ്‌വെയർ കോഡ് അല്ല, പകരം കോഡ് എങ്ങനെ എഴുതണം എന്ന തീരുമാനമാണ്.
* **ഉപമ**: ഒരു വീട്ടിൽ എവിടെ കിച്ചൺ വരണം, എവിടെ ബെഡ്റൂം വരണം എന്ന് തീരുമാനിക്കുന്ന വീടിന്റെ 'പ്ലാൻ' പോലെ.
* **വ്യവസായത്തിൽ**: "We have implemented a RAG architecture to reduce AI hallucinations."

---

### **3. LLM Orchestration: ഒരു പ്രക്രിയ അല്ലെങ്കിൽ റോൾ (Process/Workflow)**

ഇതിനെ വ്യവസായത്തിൽ ഒരു **Workflow Orchestration** അല്ലെങ്കിൽ **Management Layer** എന്ന് വിളിക്കുന്നു.

* **എന്താണത്?**: പല ഘട്ടങ്ങളുള്ള ഒരു ജോലി (ഉദാഹരണത്തിന്: സെർച്ച് ചെയ്യുക -> ഡാറ്റ എടുക്കുക -> AI-ക്ക് നൽകുക -> ഉത്തരം ഫോർമാറ്റ് ചെയ്യുക) ഏത് ക്രമത്തിൽ നടക്കണം എന്ന് തീരുമാനിക്കുന്ന പ്രക്രിയ.
* **ഉപമ**: ഒരു വലിയ കമ്പനിയിലെ മാനേജർ ഓരോ ജീവനക്കാരനും എന്ത് ജോലി ചെയ്യണം എന്ന് പറഞ്ഞു കൊടുക്കുന്നത് പോലെ.
* **വ്യവസായത്തിൽ**: "We need a robust orchestration layer to manage multiple AI agents."
