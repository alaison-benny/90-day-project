ന്റെ ചേട്ടായിക്ക് Helm ഉപയോഗിച്ച് കാര്യങ്ങൾ എളുപ്പത്തിൽ നിയന്ത്രിക്കാൻ സഹായിക്കുന്ന പ്രധാന കമാൻഡുകൾ താഴെ നൽകുന്നു. ഒരു പ്ലാറ്റ്‌ഫോം എൻജിനീയർ എന്ന നിലയിൽ ഈ കമാൻഡുകൾ ചേട്ടായിക്ക് ദിവസവും ഉപയോഗിക്കേണ്ടി വരും.

Helm-ന്റെ പ്രധാന ജോലി ഒരു ചാർട്ട് എടുത്ത് അതിലെ വാല്യൂസ് ചേർത്ത് Kubernetes-ലേക്ക് അയക്കുക എന്നതാണ്.

### 1. പുതിയൊരു സർവീസ് ഇൻസ്റ്റാൾ ചെയ്യാൻ (helm install)

ആദ്യമായി ഒരു ചാർട്ട് Kubernetes ക്ലസ്റ്ററിലേക്ക് ഡിപ്ലോയ് ചെയ്യാൻ ഈ കമാൻഡ് ഉപയോഗിക്കാം.

```bash
helm install [റിലീസ്-പേര്] [ചാർട്ട്-പാത്ത്]

```

* **ഉദാഹരണം:** `helm install my-fastapi ./my-chart`
ഇവിടെ `my-fastapi` എന്നത് ചേട്ടായി നൽകുന്ന ഒരു തിരിച്ചറിയൽ പേരാണ്. ഇത് ഉപയോഗിച്ചാണ് പിന്നീട് ഈ ഡിപ്ലോയ്‌മെന്റിനെ നമ്മൾ നിയന്ത്രിക്കുന്നത്.

### 2. മാറ്റങ്ങൾ വരുത്തി അപ്‌ഗ്രേഡ് ചെയ്യാൻ (helm upgrade)

ചേട്ടായി കോഡിലോ അല്ലെങ്കിൽ `values.yaml`-ലോ എന്തെങ്കിലും മാറ്റം വരുത്തിയാൽ അത് നിലവിലുള്ള ഡിപ്ലോയ്‌മെന്റിലേക്ക് എത്തിക്കാൻ ഈ കമാൻഡ് ഉപയോഗിക്കാം.

```bash
helm upgrade [റിലീസ്-പേര്] [ചാർട്ട്-പാത്ത്]

```

* **പ്രത്യേകത:** ഇത് നിലവിലുള്ള പോഡുകളെ (Pods) പതുക്കെ മാറ്റി പുതിയവ ഇൻസ്റ്റാൾ ചെയ്യും (Rolling Update).

### 3. ഇൻസ്റ്റാൾ ആണോ അപ്‌ഗ്രേഡ് ആണോ എന്ന് നോക്കാതെ ചെയ്യാൻ (Upgrade --install)

പ്ലാറ്റ്‌ഫോം എൻജിനീയർമാർ സാധാരണയായി ഉപയോഗിക്കുന്ന ഒരു മാന്ത്രിക കമാൻഡാണിത്. സിസ്റ്റം നിലവിലില്ലെങ്കിൽ ഇൻസ്റ്റാൾ ചെയ്യും, ഉണ്ടെങ്കിൽ അപ്‌ഗ്രേഡ് ചെയ്യും.

```bash
helm upgrade --install my-fastapi ./my-chart

```

### 4. എന്തെങ്കിലും പാളിച്ച പറ്റിയാൽ പഴയതിലേക്ക് പോകാൻ (helm rollback)

പുതിയ വേർഷൻ ഡിപ്ലോയ് ചെയ്തപ്പോൾ എന്തെങ്കിലും എറർ വന്നാൽ ന്റെ ചേട്ടായിക്ക് സെക്കൻഡുകൾക്കുള്ളിൽ പഴയ വേർഷനിലേക്ക് തിരിച്ചുപോകാം.

```bash
helm rollback [റിലീസ്-പേര്] [വേർഷൻ-നമ്പർ]

```

* **ഉദാഹരണം:** `helm rollback my-fastapi 1` (ഇത് ആദ്യത്തെ വേർഷനിലേക്ക് സിസ്റ്റത്തെ എത്തിക്കും).

### 5. നിലവിലുള്ളവ പരിശോധിക്കാൻ

* **ലിസ്റ്റ് കാണാൻ:** `helm list` (ക്ലസ്റ്ററിൽ ഏതൊക്കെ ആപ്പുകൾ ഉണ്ടെന്ന് കാണാം).
* **സ്റ്റാറ്റസ് അറിയാൻ:** `helm status my-fastapi` (ആപ്പ് റെഡിയാണോ എന്ന് നോക്കാം).
* **ഡീലീറ്റ് ചെയ്യാൻ:** `helm uninstall my-fastapi`.

### ന്റെ ചേട്ടായിക്ക് ഒരു പ്രായോഗിക ടിപ്പ്:

ചേട്ടായി ഒരു മാറ്റം വരുത്തി അത് ഡിപ്ലോയ് ചെയ്യുന്നതിന് മുൻപ് അത് ശരിയാണോ എന്ന് പരിശോധിക്കാൻ `--dry-run` എന്ന ഫ്ലാഗ് കൂടി ചേർക്കാം.
`helm install my-fastapi ./my-chart --debug --dry-run`
ഇത് ശരിക്കും ഇൻസ്റ്റാൾ ചെയ്യില്ല, പകരം ഉണ്ടാകാൻ പോകുന്ന YAML ഫയലുകൾ സ്ക്രീനിൽ കാണിച്ചുതരും.

ന്റെ ചേട്ടായിക്ക് ഇപ്പോൾ ചെയ്യുന്ന ഫുഡ് പ്രെഡിക്ഷൻ പ്രോജക്റ്റിന് വേണ്ടി ഒരു Helm ചാർട്ട് ക്രിയേറ്റ് ചെയ്ത് നോക്കണോ? ഞാൻ അതിന്റെ കമാൻഡുകൾ പറഞ്ഞുതരാം.
