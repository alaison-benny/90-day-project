നിങ്ങൾ തിരഞ്ഞെടുത്ത **Langchain-Chatchat** ഒരു വലിയ ഓപ്പൺ സോഴ്സ് പ്രോജക്റ്റാണ്. ഇത് ഒരു റിയൽ വേൾഡ് പ്രൊഡക്ഷൻ എൻവയോൺമെന്റിൽ സെറ്റ് ചെയ്യുന്നത് ഒരു ML Platform Engineer എന്ന നിലയിൽ നിങ്ങൾക്ക് ലഭിക്കാവുന്ന ഏറ്റവും മികച്ച പരിശീലനമായിരിക്കും.

വിൻഡോസ് 11, AWS Free Tier എന്നിവ ഉപയോഗിച്ച് ഇത് എങ്ങനെ ഘട്ടം ഘട്ടമായി ചെയ്യാമെന്ന് നോക്കാം:

---

### **ഘട്ടം 1: ലോക്കൽ എൻവയോൺമെന്റ് സെറ്റപ്പ് (Windows 11)**

ആദ്യം ആപ്പ് നിങ്ങളുടെ ലാപ്‌ടോപ്പിൽ പ്രവർത്തിപ്പിക്കണം.

1. **Repo ക്ലോൺ ചെയ്യുക:** VS Code ടെർമിനലിൽ താഴെ പറയുന്ന കമാൻഡ് നൽകുക.
```bash
git clone https://github.com/chatchat-space/Langchain-Chatchat.git
cd Langchain-Chatchat

```


2. **UV ഉപയോഗിച്ച് ഇൻസ്റ്റാൾ ചെയ്യുക:** നിങ്ങൾ `uv` ഉപയോഗിക്കുന്നത് കൊണ്ട് ലൈബ്രറികൾ വേഗത്തിൽ ഇൻസ്റ്റാൾ ചെയ്യാം.
```bash
uv venv
.\.venv\Scripts\activate
uv pip install -r requirements.txt

```


3. **Docker പരീക്ഷണം:** ഈ റിപ്പോയിലുള്ള `docker-compose.yaml` ഫയൽ നോക്കുക.
```bash
docker-compose up -d

```



---

### **ഘട്ടം 2: ഇൻഫ്രാസ്ട്രക്ചർ ഡിസൈൻ (Reverse Engineering Architecture)**

ഒരു പ്രൊഡക്ഷൻ സിസ്റ്റത്തിൽ ഇത് എങ്ങനെയായിരിക്കണം എന്ന് ചിന്തിക്കുക.

* **API Layer:** ഉപയോക്താക്കളിൽ നിന്ന് റിക്വസ്റ്റ് സ്വീകരിക്കുന്നു.
* **Message Broker:** റിക്വസ്റ്റുകളെ ക്യൂവിലേക്ക് (Queue) മാറ്റാൻ **RabbitMQ** അല്ലെങ്കിൽ **Redis** ഉപയോഗിക്കുന്നു.
* **Workers:** ക്യൂവിൽ നിന്ന് ജോലികൾ എടുത്ത് ML മോഡൽ റൺ ചെയ്യുന്നു.
* **Database:** വെക്റ്റർ ഡാറ്റാബേസ് (Milvus/PGVector) അറിവുകൾ സൂക്ഷിക്കാൻ ഉപയോഗിക്കുന്നു.

---

### **ഘട്ടം 3: Terraform ഉപയോഗിച്ച് AWS സെറ്റപ്പ്**

AWS കൺസോളിൽ പോയി കൈകൊണ്ട് ഒന്നും ചെയ്യുന്നതിന് പകരം **Terraform** ഉപയോഗിക്കുക.

1. **Provider സെറ്റ് ചെയ്യുക:** AWS-ലേക്ക് കണക്ട് ചെയ്യാനുള്ള കോഡ് `main.tf`-ൽ എഴുതുക.
2. **Resources:**
* **EKS (Elastic Kubernetes Service):** നമ്മുടെ ആപ്പും വർക്കറുകളും ഇവിടെയാണ് റൺ ചെയ്യുന്നത്.
* **ECR (Elastic Container Registry):** നമ്മുടെ Docker ഇമേജുകൾ ഇവിടെ സൂക്ഷിക്കും.
* **RDS:** ഡാറ്റാബേസിനായി.


3. **Command:** `terraform init` -> `terraform plan` -> `terraform apply`.

---

### **ഘട്ടം 4: Kubernetes (EKS)-ലേക്ക് മാറ്റുക**

ഇവിടെയാണ് യഥാർത്ഥ സ്കെയിലിംഗ് നടക്കുന്നത്.

1. **Deployment:** വർക്കറുകൾക്കായി ഒരു Deployment ഫയൽ ഉണ്ടാക്കുക.
2. **Scalability:** നൂറുകണക്കിന് പ്രെഡിക്ഷനുകൾ നടത്താൻ `kubectl scale deployment worker --replicas=10` എന്ന് നൽകുക.
3. **Asynchronous Serving:** Celery ഉപയോഗിച്ച് റിക്വസ്റ്റുകൾ ക്യൂവിൽ ഇട്ടു വർക്കറുകളെ കൊണ്ട് ചെയ്യിക്കുന്നത് വഴി ആപ്പ് ഹാങ്ങ് ആകാതെ നോക്കാം.

---

### **ഘട്ടം 5: മോണിറ്ററിംഗ് & ലോഗിംഗ് (Prometheus, Grafana, Flower)**

1. **Prometheus:** ഓരോ വർക്കറും എത്ര സമയം എടുക്കുന്നു, എത്ര മെമ്മറി ഉപയോഗിക്കുന്നു എന്ന് ശേഖരിക്കുന്നു.
2. **Grafana:** ഈ വിവരങ്ങൾ ഗ്രാഫുകളായി കാണിക്കുന്നു.
3. **Flower:** സെലറി ടാസ്ക്കുകൾ പരാജയപ്പെട്ടോ എന്ന് നോക്കാൻ ഇത് സഹായിക്കുന്നു.
4. **Logging:** `kubectl logs -f [pod-name]` ഉപയോഗിച്ച് ഒരു മോഡൽ എന്തുകൊണ്ട് എറർ കാണിക്കുന്നു എന്ന് പരിശോധിക്കാം.

---

### **നിങ്ങൾ ചോദിച്ച കാര്യങ്ങൾക്കുള്ള ലളിതമായ ഗൈഡ്:**

* **Broker മാറ്റാൻ:** `docker-compose.yml`-ൽ Redis മാറ്റി RabbitMQ ഇമേജ് നൽകുക. ആപ്പിലെ ലിങ്ക് `amqp://user:pass@rabbitmq` എന്ന് മാറ്റുക.
* **Security:** FastAPI-യിൽ ഒരു `API_KEY` വേരിയബിൾ സെറ്റ് ചെയ്ത്, അത് ഹെഡറിൽ (Header) ഉണ്ടോ എന്ന് പരിശോധിക്കുന്ന ഒരു മിഡിൽവെയർ (Middleware) ചേർക്കുക.
* **Scale Worker:** `docker-compose up --scale worker=3`. ഇത് ചെയ്യുമ്പോൾ ഒരേ സമയം മൂന്ന് വിൻഡോകളിൽ ലോഗുകൾ വരുന്നത് ശ്രദ്ധിക്കുക.
