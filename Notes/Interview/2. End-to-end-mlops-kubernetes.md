## **The "Elevator Pitch" (Brief Summary)**
"I built an end-to-end MLOps platform on AWS to deploy and manage a Sentiment Analysis microservice. ## **Detailed Project Breakdown (The "STAR" Method)**

### **1. Infrastructure & Provisioning**
"To ensure the infrastructure was reproducible and scalable, I used **Terraform** to provision a managed **AWS EKS** cluster. I didn't just stop at the cluster; I configured the VPC networking, IAM roles for service accounts (IRSA), and optimized node groups to balance performance and cost."


### **2. Application Packaging & Deployment**
"I containerized a **FastAPI**-based ML model using Docker. For the deployment phase, I transitioned from standard Kubernetes manifests to **Helm Charts**. This allowed me to manage repeatable deployments with environment-specific overrides (values.yaml), making it easy to roll back or upgrade the application version in seconds."

### **3. Traffic Management & Scaling**
"I implemented an **AWS ALB Ingress Controller** to handle external traffic with SSL termination. To ensure the platform could handle production-level spikes, I configured a **Horizontal Pod Autoscaler (HPA)**. During this phase, I solved critical issues like metrics-server misconfigurations and fine-tuned resource limits (requests/limits) to prevent OOMKills."


### **4. Observability & Debugging**
"For a platform to be production-ready, it needs monitoring. I deployed the **Prometheus Operator and Grafana** to create observability dashboards. This helped me debug issues like scrape config failures and missing metrics, ensuring I had a 360-degree view of both cluster health and model API performance."

### **5. Automation (CI/CD)**
"Finally, I automated the whole flow using **GitHub Actions**. Every time I push code, the pipeline builds the Docker image, pushes it to the registry, and triggers a Helm upgrade on the EKS cluster, ensuring a seamless deployment process."
