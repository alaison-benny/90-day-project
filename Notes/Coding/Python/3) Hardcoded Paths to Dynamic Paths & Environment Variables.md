###  Consept 1) Hardcoded Paths മാറ്റുക Using Dynamic Paths & Environment Variables

നമസ്കാരം ചേട്ടായി! നമുക്ക് ഈ കോൺസെപ്റ്റ് വളരെ ലളിതമായി പഠിക്കാം. ചേട്ടായിക്ക് മനസ്സിലാക്കാൻ പ്രയാസമുള്ള ആ ഭാഗം—**"പേരുകൾ എങ്ങനെ തീരുമാനിക്കുന്നു?"** എന്നതിൽ നിന്ന് തന്നെ തുടങ്ങാം.

---

### 1. ഈ പേരുകൾ എവിടെ നിന്ന് വരുന്നു? (Where do these names come from?)

ഇത് മനസ്സിലാക്കാൻ ഒരു ഉദാഹരണം പറയാം. ചേട്ടായി ഒരു കട നടത്തുകയാണെന്ന് വിചാരിക്കുക. കടയിലെ സാധനങ്ങൾ വെക്കാൻ ചേട്ടായി കുറച്ച് പെട്ടികൾ വാങ്ങി.
* ഒരു പെട്ടിയിൽ **"പഞ്ചസാര"** എന്ന് എഴുതി.
* മറ്റൊന്നിൽ **"ചായപ്പൊടി"** എന്ന് എഴുതി.

ഇവിടെ **"പഞ്ചസാര"**, **"ചായപ്പൊടി"** എന്നീ പേരുകൾ പൈത്തൺ നൽകിയതല്ല, പകരം ചേട്ടായി (Platform Engineer) നൽകിയതാണ്. അതുപോലെ കോഡിൽ നമ്മൾ ഉപയോഗിക്കുന്ന `MODEL_PATH`, `DATABASE_URL` തുടങ്ങിയവ നമ്മൾ ഇടുന്ന പേരുകളാണ്. സർവറിലെ ആ പെട്ടിയുടെ മുകളിലുള്ള **ലേബൽ (Label)** ആണിത്.



---

### 2. എങ്ങനെ മാറ്റണം? (Step-by-Step)

**സ്റ്റെപ്പ് 1: പഴയ കോഡ് നോക്കുക**
ഡാറ്റാ സയന്റിസ്റ്റ് തന്ന കോഡിൽ ഇങ്ങനെ ഉണ്ടാകും:
`MODEL_PATH = "/home/ubuntu/mlruns/1/model"`
(ഇവിടെ `/home/ubuntu/...` എന്നത് അയാൾ അയാളുടെ വീട്ടിലെ അഡ്രസ്സ് എഴുതിയത് പോലെയാണ്. അത് മാറ്റണം).

**സ്റ്റെപ്പ് 2: ഒരു പേര് തീരുമാനിക്കുക (The Label)**
ഈ സാധനം എന്താണ്? ഇതൊരു 'Model' ആണ്. അപ്പോൾ നമുക്ക് `MODEL_PATH` എന്ന് പേരിടാം. (എപ്പോഴും ക്യാപിറ്റൽ ലെറ്ററിൽ എഴുതുക).

**സ്റ്റെപ്പ് 3: ഡിഫോൾട്ട് വഴി നൽകുക (The Backup)**
സർവറിൽ ആ പേര് കണ്ടില്ലെങ്കിൽ എന്ത് ചെയ്യണം? ചേട്ടായിയുടെ കമ്പ്യൂട്ടറിലെ വഴി നൽകുക: `./models/my_model`

**സ്റ്റെപ്പ് 4: കോഡ് എഴുതുക**
`MODEL_PATH = os.getenv("MODEL_PATH", "./models/my_model")`

### 3. നാല് ഉദാഹരണങ്ങൾ (4 Real-Life Examples)

ഓരോ കാര്യത്തിനും എന്ത് പേര് നൽകണം എന്ന് താഴെ നോക്കാം:

#### **ഉദാഹരണം 1: ഡാറ്റാബേസ് (Database)**
* **എന്ത് മാറ്റണം:** `db_file = "C:/Users/chittayi/data.db"`
* **എങ്ങനെ മാറ്റണം:**
```python
import os
# പേര്: DATABASE_PATH (നമ്മൾ തീരുമാനിച്ചത്)
# ഡിഫോൾട്ട്: "local.db"
db_file = os.getenv("DATABASE_PATH", "local.db")
```
* **വിശദീകരണം:** സർവറിൽ `DATABASE_PATH` എന്ന പെട്ടി ഉണ്ടോ എന്ന് നോക്കും. ഉണ്ടെങ്കിൽ അതിലെ അഡ്രസ്സ് എടുക്കും, ഇല്ലെങ്കിൽ `local.db` ഉപയോഗിക്കും.

#### **ഉദാഹരണം 2: ലോഗ് ഫയലുകൾ (Logs)**
* **എന്ത് മാറ്റണം:** `log_loc = "/var/logs/my_app.log"`
* **എങ്ങനെ മാറ്റണം:**
```python
import os
# പേര്: APP_LOGS
# ഡിഫോൾട്ട്: "app.log"
log_loc = os.getenv("APP_LOGS", "app.log")
```
* **വിശദീകരണം:** ആപ്പിന്റെ തെറ്റുകൾ രേഖപ്പെടുത്തുന്ന ഫയൽ എവിടെ വേണമെന്ന് സർവർ സെറ്റിംഗ്സ് വഴി മാറ്റാൻ പ്ലാറ്റ്‌ഫോം എൻജിനീയറെ ഇത് സഹായിക്കുന്നു.

#### **ഉദാഹരണം 3: ഫോട്ടോകൾ സേവ് ചെയ്യാൻ (Images)**
* **എന്ത് മാറ്റണം:** `folder = "D:/projects/images"`
* **എങ്ങനെ മാറ്റണം:**
```python
import os
# പേര്: UPLOAD_FOLDER
# ഡിഫോൾട്ട്: "./uploads"
folder = os.getenv("UPLOAD_FOLDER", "./uploads")
```
* **വിശദീകരണം:** യൂസർമാർ അയക്കുന്ന ഫോട്ടോകൾ എവിടെ സൂക്ഷിക്കണമെന്ന് ഈ വരി തീരുമാനിക്കുന്നു.

#### **ഉദാഹരണം 4: കോഡ് ഇരിക്കുന്ന സ്ഥലം (Code Path)**
ഇവിടെ സർവറിനോട് ചോദിക്കാതെ, പൈത്തണിനോട് തന്നെ "നീ എവിടെയാണ് ഇരിക്കുന്നത്" എന്ന് ചോദിക്കുന്നു.
```python
import os
# __file__ എന്നാൽ നിലവിലെ ഈ ഫയൽ എന്നാണ് അർത്ഥം
# os.path.abspath അത് ഇരിക്കുന്ന മുഴുവൻ അഡ്രസ്സും തരുന്നു
BASE_DIR = os.path.dirname(os.path.abspath(__file__))

# ഇനി ഈ ഫോൾഡറിനുള്ളിലെ 'config' എന്ന ഫയൽ എടുക്കാൻ:
CONFIG_PATH = os.path.join(BASE_DIR, "config.json")
```
* **വിശദീകരണം:** ഇത് സർവറിനെ ആശ്രയിക്കുന്നില്ല. കോഡ് എവിടെയാണോ ഇരിക്കുന്നത്, അതിന്റെ കൂടെയുള്ള ഫയലുകൾ എടുക്കാൻ ഇത് സഹായിക്കും.

### 4. എപ്പോൾ മാറ്റണം? എങ്ങനെ കണ്ടുപിടിക്കാം?

* **എപ്പോൾ മാറ്റണം:** ന്റെ ചേട്ടായി എഴുതിയ കോഡ് തന്റെ ലാപ്‌ടോപ്പിന് പുറത്തേക്ക് (ഉദാഹരണത്തിന് മറ്റൊരു സുഹൃത്തിനോ സർവറിനോ) കൊടുക്കാൻ പോകുമ്പോൾ ഇത് മാറ്റണം.
* **എങ്ങനെ കണ്ടുപിടിക്കാം:** കോഡിൽ എവിടെയെങ്കിലും ഇതുപോലെ കണ്ടാൽ ഉടൻ മാറ്റണം:
    1.  `C:/` അല്ലെങ്കിൽ `/home/` എന്ന് തുടങ്ങുന്ന അഡ്രസ്സുകൾ.
    2.  യൂസർ നെയിമുകൾ (ഉദാ: `/Users/praveen/...`).
    3.  ഫയലുകളുടെ പേര് നേരിട്ട് എഴുതിയിരിക്കുന്നത്.

### ഓരോ കാര്യത്തിനും ഉപയോഗിക്കുന്ന പൊതുവായ പേരുകൾ (Bullet Points)

ന്റെ ചേട്ടായിക്ക് എളുപ്പത്തിൽ പഠിക്കാൻ താഴെ നൽകുന്നു:

* **മോഡൽ പാത്ത്:** `MODEL_PATH`, `MODEL_DIR`
* **ഡാറ്റാബേസ്:** `DATABASE_URL`, `DB_CONFIG`
* **ലോഗുകൾ:** `LOG_PATH`, `LOG_FILE`
* **ഇമേജുകൾ/ഫയലുകൾ:** `STORAGE_DIR`, `UPLOAD_PATH`
* **സീക്രട്ട് കീകൾ:** `API_KEY`, `AUTH_TOKEN` (ഇവയ്ക്ക് ഒരിക്കലും രണ്ടാമത്തെ അഡ്രസ്സ്/ഡിഫോൾട്ട് നൽകരുത്).

## കോൺസെപ്റ്റ് 1: Hardcoded Paths മാറ്റുക (Using Dynamic Paths & Environment Variables)

### എന്താണ് ഈ കോൺസെപ്റ്റ്? (Deep Theory)
ഒരു ഡാറ്റാ സയന്റിസ്റ്റ് തന്റെ കമ്പ്യൂട്ടറിൽ (Laptop) കോഡ് എഴുതുമ്പോൾ മോഡൽ ഇരിക്കുന്ന സ്ഥലം ഇങ്ങനെ എഴുതി വെക്കും: 
`MODEL_PATH = "/home/user/desktop/my_model"`. ഇതിനെയാണ് **Hardcoding** എന്ന് പറയുന്നത്.

**പ്രശ്നം:** ന്റെ ചേട്ടായി ഈ കോഡ് എടുത്ത് ഒരു ക്ലൗഡ് സർവറിലോ (AWS/Azure) അല്ലെങ്കിൽ ഒരു ഡോക്കർ കണ്ടെയ്‌നറിലോ ഇടുമ്പോൾ അവിടെ ഇങ്ങനെയൊരു പാത്ത് (Path) ഉണ്ടാകണമെന്നില്ല. അപ്പോൾ ആപ്പ് ക്രാഷ് ആകും.

**പരിഹാരം:** പാത്തുകൾ ഡൈനാമിക് (Dynamic) ആക്കണം. അതായത്, കോഡ് റൺ ചെയ്യുന്ന സ്ഥലം നോക്കി സ്വയം പാത്ത് കണ്ടുപിടിക്കണം അല്ലെങ്കിൽ പുറമെ നിന്ന് ഒരു വാല്യൂ ആയി നൽകണം. ഇതിന് `os` ലൈബ്രറിയിലെ `os.getenv()` അല്ലെങ്കിൽ `os.path` രീതികൾ ഉപയോഗിക്കുന്നു.



---

### എങ്ങനെ മാറ്റണം? (How to write it)

1.  **ഘട്ടം 1:** ആദ്യം `import os` ചേർക്കുക.
2.  **ഘട്ടം 2:** സ്ഥിരമായ പാത്ത് മാറ്റി `os.getenv("VARIABLE_NAME", "DEFAULT_VALUE")` ഉപയോഗിക്കുക. 
3.  **ഘട്ടം 3:** അല്ലെങ്കിൽ നിലവിലെ ഫയൽ ഇരിക്കുന്ന ഫോൾഡർ ബേസ് ആയി എടുക്കാൻ `os.path.dirname(os.path.abspath(__file__))` ഉപയോഗിക്കുക.

---

### 4 ഉദാഹരണങ്ങൾ (4 Examples)

#### ഉദാഹരണം 1: `os.getenv` ഉപയോഗിച്ച് (ഏറ്റവും മികച്ച രീതി)
```python
import os

# പഴയ രീതി: MODEL_PATH = "/home/ubuntu/mlruns/1/model"
# പുതിയ രീതി:
MODEL_PATH = os.getenv("MODEL_PATH", "/app/models/default_model")

print(f"മോഡൽ എടുക്കുന്നത് ഇവിടെ നിന്നാണ്: {MODEL_PATH}")
```
* **വിശകലനം:** ഇവിടെ `os.getenv` ആദ്യം സർവറിലെ 'MODEL_PATH' എന്ന സെറ്റിംഗ്സ് നോക്കും. അത് ഇല്ലെങ്കിൽ മാത്രം രണ്ടാമത് കൊടുത്ത `/app/models/default_model` എടുക്കും. ഇത് പ്ലാറ്റ്‌ഫോം എൻജിനീയർമാർക്ക് സർവർ സെറ്റിംഗ്സ് വഴി പാത്ത് മാറ്റാൻ സൗകര്യം നൽകുന്നു.

#### ഉദാഹരണം 2: `os.path` ഉപയോഗിച്ച് (ലോക്കൽ ഫയലുകൾക്ക്)
```python
import os

# കോഡ് ഇരിക്കുന്ന ഫോൾഡർ കണ്ടുപിടിക്കുന്നു
BASE_DIR = os.path.dirname(os.path.abspath(__file__))

# ആ ഫോൾഡറിനുള്ളിലെ 'data' എന്ന സബ്-ഫോൾഡറിലേക്ക് പാത്ത് ഉണ്ടാക്കുന്നു
DATA_PATH = os.path.join(BASE_DIR, "data", "dataset.csv")

print(f"ഡാറ്റാ സെറ്റ് പാത്ത്: {DATA_PATH}")
```
* **വിശകലനം:** ചേട്ടായിയുടെ ലാപ്‌ടോപ്പിലായാലും സർവറിലായാലും ഈ കോഡ് ആ ഫയൽ ഇരിക്കുന്ന സ്ഥലം കൃത്യമായി കണ്ടുപിടിക്കും. നമ്മൾ നേരിട്ട് ടൈപ്പ് ചെയ്യേണ്ടി വരുന്നില്ല.

#### ഉദാഹരണം 3: MLflow മോഡൽ ലോഡിംഗ് (ചേട്ടായി തന്ന സാമ്പിൾ കോഡിൽ വരുത്തേണ്ട മാറ്റം)
```python
import os
import mlflow.sklearn

# പ്രൊഡക്ഷൻ റെഡി പാത്ത്
REMOTE_PATH = os.getenv("MLFLOW_MODEL_URI", "models:/my_model/Production")

try:
    model = mlflow.sklearn.load_model(REMOTE_PATH)
    print("വിജകരമായി ലോഡ് ചെയ്തു")
except Exception as e:
    print(f"Error: {e}")
```
* **വിശകലനം:** ഇവിടെ മോഡൽ ഐഡി കോഡിനുള്ളിൽ എഴുതാതെ `MLFLOW_MODEL_URI` വഴി വരുന്നു. ഇത് വേറെ വേറെ മോഡലുകൾ എളുപ്പത്തിൽ ടെസ്റ്റ് ചെയ്യാൻ സഹായിക്കും.

#### ഉദാഹരണം 4: ലോഗ് ഫയലുകൾ സൂക്ഷിക്കാൻ
```python
import os

LOG_DIR = os.getenv("LOG_DIRECTORY", "./logs")

# ഫോൾഡർ ഇല്ലെങ്കിൽ ഉണ്ടാക്കുന്നു
if not os.path.exists(LOG_DIR):
    os.makedirs(LOG_DIR)

LOG_FILE = os.path.join(LOG_DIR, "app.log")
print(f"ലോഗുകൾ ഇവിടെ സേവ് ചെയ്യും: {LOG_FILE}")
```
* **വിശകലനം:** സർവർ മാറുമ്പോൾ ലോഗ്സ് എവിടെ വയ്ക്കണം എന്ന് പ്ലാറ്റ്‌ഫോം എൻജിനീയർക്ക് തീരുമാനിക്കാം. കോഡിനുള്ളിൽ മാറ്റം വരുത്തേണ്ടി വരില്ല.

---

### എപ്പോൾ മാറ്റണം? (When to change)
* കോഡ് ഡെവലപ്‌മെന്റ് കഴിഞ്ഞ് ടെസ്റ്റിംഗിലേക്കോ പ്രൊഡക്ഷനിലേക്കോ മാറ്റുമ്പോൾ.
* കോഡ് ഒരു **Docker Container** ആക്കി മാറ്റാൻ തുടങ്ങുമ്പോൾ.
* ക്ലൗഡ് സർവീസുകളിലേക്ക് (AWS S3, Azure Blob) ഫയലുകൾ മാറ്റുമ്പോൾ.

### എങ്ങനെ മാറ്റണം? (How to find where to change)
ചേട്ടായിയുടെ കോഡിൽ കണ്ണ് ഓടിച്ചു നോക്കുക. എവിടെയെങ്കിലും ഇതുപോലെ കണ്ടാൽ ഉടൻ മാറ്റണം:
* `C:\Users\...` അല്ലെങ്കിൽ `/home/...` എന്ന് തുടങ്ങുന്ന വരികൾ.
* ഫയൽ പാത്തുകൾ സ്ട്രിംഗ് ആയി (`" "`) നേരിട്ട് നൽകിയിരിക്കുന്നത്.
* യൂസർ നെയിമുകൾ പാത്തിൽ അടങ്ങിയിട്ടുണ്ടെങ്കിൽ.
