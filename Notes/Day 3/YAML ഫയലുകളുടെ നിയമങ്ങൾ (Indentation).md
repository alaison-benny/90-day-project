# KW: Teach me YAML ഫയലുകളുടെ നിയമങ്ങൾ (Indentation) in simple language and real world production examples
നമുക്ക് **YAML** (Yet Another Markup Language) നിയമങ്ങൾ വളരെ ലളിതമായി പഠിക്കാം. ഒരു MBA ഗ്രാജുവേറ്റ് എന്ന നിലയിൽ ഇതിനെ ഒരു കമ്പനിയുടെ **Organization Hierarchy** (അധികാര ശ്രേണി) പോലെ കാണുന്നതാകും ഏറ്റവും എളുപ്പം.

### 1. ഇൻഡന്റേഷൻ (Indentation) എന്നാൽ എന്ത്?

YAML-ൽ ടാബുകൾ (Tabs) ഉപയോഗിക്കാൻ പാടില്ല. പകരം **സ്പേസുകൾ (Spaces)** മാത്രമേ ഉപയോഗിക്കാവൂ. സാധാരണയായി **രണ്ട് സ്പേസുകൾ** ആണ് ഒരു ലെവൽ മാറ്റാൻ ഉപയോഗിക്കുന്നത്.

ഇതൊരു ഫാമിലി ട്രീ പോലെയാണ്. മുകളിലുള്ളത് പാരന്റ് (Parent), താഴെ സ്പേസ് ഇട്ട് നിൽക്കുന്നത് ചൈൽഡ് (Child).

---

### 2. YAML-ലെ പ്രധാന നിയമങ്ങൾ

#### **A. പാരന്റ്-ചൈൽഡ് ബന്ധം (Hierarchy)**

ഒരു വരിക്ക് താഴെ സ്പേസ് ഇട്ട് അടുത്ത വരി എഴുതിയാൽ, രണ്ടാമത്തെ വരി ആദ്യത്തേതിന്റെ ഭാഗമാണെന്ന് അർത്ഥം.

**Real-world Example (Office Structure):**

```yaml
Company:
  CEO: "Alaison"        # രണ്ട് സ്പേസ് ഉള്ളിൽ - CEO കമ്പനിയുടെ കീഴിലാണ്
  Departments:          # രണ്ട് സ്പേസ് ഉള്ളിൽ
    IT_Department:      # നാല് സ്പേസ് ഉള്ളിൽ - ഇത് ഡെപ്പാർട്ട്മെന്റിന്റെ ഉള്ളിലാണ്
      Lead: "Benny"     # ആറ് സ്പേസ്

```

#### **B. ലിസ്റ്റുകൾ (Lists/Arrays)**

ഒരേപോലെയുള്ള ഒന്നിലധികം കാര്യങ്ങൾ പറയാൻ ഹൈഫൻ (`-`) ഉപയോഗിക്കുന്നു.

**Example:**

```yaml
Programming_Languages:
  - Python
  - JavaScript
  - Go

```

#### **C. കീ-വാല്യൂ ജോഡികൾ (Key-Value Pairs)**

എപ്പോഴും ഒരു പേരും (Key) അതിന്റെ വാല്യൂവും (Value) തമ്മിൽ ഒരു കോളം (`:`) ഉണ്ടാകണം. കോളത്തിന് ശേഷം ഒരു **സ്പേസ് നിർബന്ധമാണ്**.

* ശരി: `name: Alaison`
* തെറ്റ്: `name:Alaison`

### 3. പ്രൊഡക്ഷനിലെ ഉദാഹരണം (Docker Compose)

നമ്മുടെ പ്രോജക്റ്റിലെ `docker-compose.yaml` എടുത്തു നോക്കിയാൽ ഈ നിയമങ്ങൾ കാണാം:

```yaml
version: '3.8'          # പാരന്റ് 1 (ഏറ്റവും പുറത്ത്)

services:               # പാരന്റ് 2
  api:                  # ചൈൽഡ് ഓഫ് 'services' (2 സ്പേസ്)
    image: python:3.9   # ചൈൽഡ് ഓഫ് 'api' (4 സ്പേസ്)
    ports:              # ചൈൽഡ് ഓഫ് 'api'
      - "8000:8000"     # ലിസ്റ്റ് ഐറ്റം (ports-ന്റെ കീഴിൽ)
  
  db:                   # api-ക്ക് നേരെ (അതായത് db മറ്റൊരു സർവീസ് ആണ്)
    image: redis:latest

```

---

### 4. ശ്രദ്ധിക്കേണ്ട 3 ഗോൾഡൻ നിയമങ്ങൾ

1. **നേർരേഖ പാലിക്കുക:** ഒരേ ലെവലിലുള്ള കാര്യങ്ങൾ കൃത്യം ഒരേ ലൈനിൽ വരണം. `api:` ഉം `worker:` ഉം ഒരേ വരിയിൽ വരുന്നത് പോലെ.
2. **Tabs ഉപയോഗിക്കരുത്:** നിങ്ങളുടെ കീബോർഡിലെ `Tab` കീ YAML ഫയലുകളിൽ ഉപയോഗിക്കരുത്. എപ്പോഴും `Space` ഉപയോഗിക്കുക.
3. **കോളത്തിന് ശേഷം സ്പേസ്:** `:` കഴിഞ്ഞാൽ ഒരു സ്പേസ് ഇട്ടിട്ടേ വാല്യൂ എഴുതാവൂ.

### ഒരു സിമ്പിൾ ടിപ്പ്:

നിങ്ങൾ എഴുതിയ YAML ശരിയാണോ എന്ന് പരിശോധിക്കാൻ **"YAML Lint"** പോലുള്ള ഓൺലൈൻ ടൂളുകൾ ലഭ്യമാണ്. തെറ്റുണ്ടെങ്കിൽ അത് ചുവന്ന അടയാളം കാണിക്കും.

നമ്മുടെ `docker-compose.yaml` ഫയലിൽ **Flower** ആഡ് ചെയ്യുമ്പോൾ ഈ സ്പേസുകൾ ശ്രദ്ധിക്കണം. നമുക്ക് ഇപ്പോൾ അത് എഡിറ്റ് ചെയ്യാൻ തുടങ്ങിയാലോ? അതോ ഇതിൽ കൂടുതൽ വ്യക്തത വേണോ?
