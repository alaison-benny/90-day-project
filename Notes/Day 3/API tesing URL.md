ഈ യുആർഎൽ (URL) എവിടെ നിന്ന് വന്നു എന്നത് വളരെ പ്രധാനപ്പെട്ട ഒരു ചോദ്യമാണ്. ഇത് നമ്മൾ തന്നെ **`main.py`** എന്ന ഫയലിൽ എഴുതിവെച്ച കോഡിൽ നിന്നാണ് ഉണ്ടായത്.

ഒരു ആപ്ലിക്കേഷൻ നിർമ്മിക്കുമ്പോൾ ഡെവലപ്പർമാർ തീരുമാനിക്കുന്ന വഴികളെയാണ് **Endpoints** എന്ന് വിളിക്കുന്നത്. അത് എങ്ങനെയാണ് നമ്മൾ കണ്ടുപിടിക്കുന്നത് എന്ന് താഴെ വിവരിക്കാം:

### 1. നമ്മൾ എഴുതിയ കോഡ് പരിശോധിക്കുക (The Source)

നിങ്ങൾ `main.py` എന്ന ഫയലിൽ താഴെ പറയുന്ന വരികൾ എഴുതിയിട്ടുണ്ട്:

* **`@app.post("/analyze")`**: ഇതിനർത്ഥം ഡാറ്റ അയക്കാൻ `/analyze` എന്ന പേര് ഉപയോഗിക്കണം എന്നാണ്.
* **`def trigger_analysis(text: str):`**: ഇതിനർത്ഥം അയക്കുന്ന ഡാറ്റയുടെ കൂടെ `text` എന്ന പേരിൽ ഒരു വാചകം കൂടി ചേർക്കണം എന്നാണ്.

ഇതുകൊണ്ടാണ് URL ഇങ്ങനെ വന്നത്: `http://<IP>:8000/analyze?text=ഹലോ`.

### 2. ഓട്ടോമാറ്റിക് ഡോക്യുമെന്റേഷൻ (The Easy Way)

FastAPI-യുടെ ഏറ്റവും വലിയ പ്രത്യേകത അത് നിങ്ങൾക്കായി ഒരു **Manual** തനിയെ ഉണ്ടാക്കും എന്നതാണ്. ഇത് നോക്കിയാൽ നിങ്ങൾക്ക് എല്ലാ ടെസ്റ്റിംഗ് ലിങ്കുകളും മനസ്സിലാക്കാം.

* നിങ്ങളുടെ ബ്രൗസർ തുറന്ന് ഇത് ടൈപ്പ് ചെയ്യുക: `http://44.202.154.134:8000/docs`.
* ഇവിടെ നിങ്ങൾക്ക് നമ്മൾ ഉണ്ടാക്കിയ എല്ലാ ലിങ്കുകളും (Endpoints), അത് എങ്ങനെ ഉപയോഗിക്കണം എന്നതും ഭംഗിയായി കാണാം. ഇതിനെ **Swagger UI** എന്ന് വിളിക്കുന്നു.

### 3. URL-ന്റെ ഭാഗങ്ങൾ മനസ്സിലാക്കാം

* **`http://44.202.154.134`**: ഇത് നിങ്ങളുടെ AWS സർവറിന്റെ വിലാസമാണ്.
* **`:8000`**: ഇത് നമ്മൾ `docker-compose.yaml`-ൽ API-ക്ക് വേണ്ടി മാറ്റിവെച്ച വാതിലാണ് (Port).
* **`/analyze`**: ഇത് നമ്മൾ കോഡിൽ കൊടുത്ത പേര്.
* **`?text=...`**: ഇത് നമ്മൾ AI-ക്ക് വിശകലനം ചെയ്യാൻ കൊടുക്കുന്ന വാചകം.

### ഇനി എങ്ങനെ ടെസ്റ്റ് ചെയ്യണം?

1. **Postman** തുറക്കുക.
2. മുകളിൽ പറഞ്ഞ URL കോപ്പി ചെയ്ത് പേസ്റ്റ് ചെയ്യുക.
3. **Send** അമർത്തുക.

ഇപ്പോൾ `/docs` എന്ന ലിങ്ക് ബ്രൗസറിൽ തുറക്കാൻ സാധിക്കുന്നുണ്ടോ? അവിടെ നമ്മൾ സെറ്റ് ചെയ്ത എല്ലാ കാര്യങ്ങളും കാണാൻ പറ്റും. ഒന്ന് പരീക്ഷിച്ചു നോക്കൂ! അതിനുശേഷം നമുക്ക് പോസ്റ്റ്മാനിലേക്ക് പോകാം.
