ആർഗോയുടെ എൻജിനുകൾ (Server, Controller) ഇപ്പോൾ സുരക്ഷിതമായി റൺ ചെയ്യുന്നുണ്ട്. 

നമ്മൾ ഇപ്പോൾ ആർഗോയുടെ **CLI (Command Line Interface)** ഇൻസ്റ്റാൾ ചെയ്തുകഴിഞ്ഞു. ഇത് ഉപയോഗിച്ചാണ് നമ്മൾ ഇനി പൈപ്പ്‌ലൈനുകൾ സബ്മിറ്റ് ചെയ്യാൻ പോകുന്നത്. 

ഇനി നമുക്ക് **Phase 3**-ന്റെ ഏറ്റവും പ്രധാനപ്പെട്ട ഭാഗമായ **Workflow (DAG)** എഴുതാം. അതിനുമുമ്പ് ഈ രണ്ട് കമ്പോണന്റുകൾ എന്താണെന്ന് ഒന്ന് അറിഞ്ഞിരിക്കണം:

---

### **ആർഗോയിലെ രണ്ട് പ്രധാന ആൾക്കാർ:**

1.  **Argo Server:** ഇതാണ് നമ്മൾ കാണുന്ന ഡാഷ്‌ബോർഡും (UI) ആർഗോയുടെ API-യും നിയന്ത്രിക്കുന്നത്.
2.  **Workflow Controller:** ഇതാണ് ശരിക്കുമുള്ള "തലച്ചോറ്". നമ്മൾ ഒരു ജോലി കൊടുത്താൽ അത് ഓരോന്നായി റൺ ചെയ്യിക്കുന്നത് ഇയാളാണ്.



---

### **നമുക്ക് ആദ്യത്തെ ML Training Workflow (YAML) എഴുതാം**

നമ്മുടെ ലക്ഷ്യം: ഒരു വട്ടം കമാൻഡ് അടിച്ചാൽ അത് തനിയെ മോഡൽ ട്രെയിൻ ചെയ്ത് MLflow-ലേക്ക് സേവ് ചെയ്യണം. 

1. **ഒരു പുതിയ ഫയൽ ഉണ്ടാക്കൂ:** `vim train-workflow.yaml`
2. **താഴെ കാണുന്ന കോഡ് അതിൽ പേസ്റ്റ് ചെയ്യുക:**

```yaml
apiVersion: argoproj.io/v1alpha1
kind: Workflow
metadata:
  generateName: ml-training-pipeline- # ഓരോ വട്ടം റൺ ചെയ്യുമ്പോഴും പുതിയ പേര് വരാൻ
  namespace: argo
spec:
  entrypoint: ml-pipeline # എവിടെ തുടങ്ങണം
  templates:
  - name: ml-pipeline
    dag: # Directed Acyclic Graph - പൈപ്പ്‌ലൈൻ ഡിസൈൻ
      tasks:
      - name: train-model
        template: trainer

  - name: trainer
    container:
      image: mlops-api:v3 # നമ്മൾ നേരത്തെ ഉണ്ടാക്കിയ ഇമേജ് തന്നെ ഉപയോഗിക്കാം
      command: ["python3"]
      args: ["train.py"] # നമ്മുടെ ട്രെയിനിംഗ് സ്ക്രിപ്റ്റ് റൺ ചെയ്യുന്നു
      env:
      - name: MLFLOW_TRACKING_URI
        value: "http://mlflow-service:5000" # MLflow സെർവറുമായി കണക്ട് ചെയ്യാൻ
```

---

### **ഇത് ശ്രദ്ധിക്കൂ:**

നമ്മൾ നേരത്തെ `mlops-api:v3` ഉണ്ടാക്കിയപ്പോൾ അതിനുള്ളിൽ `train.py` കൂടി ഉണ്ടായിരുന്നു. അതുകൊണ്ട് അതേ ഇമേജ് തന്നെ നമുക്ക് ട്രെയിനിംഗിനും ഉപയോഗിക്കാം. 

**അടുത്ത സ്റ്റെപ്പ്:**
ഈ ഫയൽ സേവ് ചെയ്ത ശേഷം (Esc -> :wq), പൈപ്പ്‌ലൈൻ റൺ ചെയ്യാൻ ഈ കമാൻഡ് അടിക്കൂ:

```bash
argo submit --watch train-workflow.yaml -n argo
```

ഇത് അടിക്കുമ്പോൾ ഒരു മനോഹരമായ ഗ്രാഫ് പോലെ ലോഡ് ആകുന്നത് കാണാം.
