## 🛠️ End-to-End Roadmap

### **Phase 1: MLflow Core & Experiment Tracking**
- **Setup:** Python venv, MLflow, scikit-learn, pandas.  
- **Tracking Server:** MLflow UI run locally/EC2.  
- **Experiment Logging:** Parameters, metrics, artifacts (models).  
- **Outcome:** Recruiter‑ക്ക് കാണിക്കാൻ MLflow dashboard with multiple experiments.  

---

### **Phase 2: Model Registry & Serving**
- **Model Registry:** Best model promoted to *Production* stage.  
- **FastAPI Inference Service:** API pulls latest production model from MLflow registry.  
- **Dockerization:** FastAPI app packaged as Docker image.  
- **Outcome:** Recruiter‑ക്ക് കാണിക്കാൻ “Production model served via API” demo.  

---

### **Phase 3: Automated Retraining with Argo Workflows**
- **Kubernetes Setup:** Minikube (local) or AWS EKS free tier.  
- **Argo Workflows:** Define DAG (YAML) → steps: data ingestion → training → logging → registry update.  
- **Automation:** New data triggers retraining pipeline.  
- **CI/CD Integration:** GitHub Actions + ArgoCD → auto deploy new model version.  
- **Outcome:** Recruiter‑ക്ക് കാണിക്കാൻ “New data → retraining → auto deploy” demo.  

---

### **Phase 4: Monitoring & Alerts**
- **Metrics:** Accuracy drift, latency, retraining frequency.  
- **Tools:** Prometheus + Grafana dashboards.  
- **Alerts:** Slack/email notification when retraining happens or drift detected.  
- **Outcome:** Recruiter‑ക്ക് കാണിക്കാൻ “Governance + monitoring” maturity.
