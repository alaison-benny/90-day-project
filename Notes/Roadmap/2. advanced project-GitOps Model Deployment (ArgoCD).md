### **Project #2: GitOps Model Deployment (ArgoCD)**

ഈ പ്രോജക്റ്റിന്റെ പ്രധാന ലക്ഷ്യം ഇതാണ്: **"Git-ൽ നമ്മൾ ഒരു മാറ്റം വരുത്തിയാൽ, ഒരു കമാൻഡ് പോലും അടിക്കാതെ കുബർനെറ്റീസിലെ ആപ്പ് തനിയെ അപ്‌ഡേറ്റ് ആകണം."**

---

ഇതിന്റെ **End-to-End Workflow** താഴെ പറയുന്നതാണ്:

#### **Phase 1: Local / Cluster Setup (ദിവസം 1-2)**
പണം കളയാതിരിക്കാൻ നമുക്ക് ഇത് സ്വന്തം ലാപ്ടോപ്പിലോ അല്ലെങ്കിൽ AWS ഫ്രീ ടയറിലെ ഒരു വലിയ ഇൻസ്റ്റൻസിലോ (t3.medium ആണ് നല്ലത്, പക്ഷേ ശ്രദ്ധിക്കണം) ചെയ്യാം.
1.  **Kubernetes ഇൻസ്റ്റാൾ ചെയ്യുക:** `Minikube` അല്ലെങ്കിൽ `K3s` ഉപയോഗിച്ച് ഒരു ലോക്കൽ ക്ലസ്റ്റർ ഉണ്ടാക്കുക.
2.  **ArgoCD ഇൻസ്റ്റാൾ ചെയ്യുക:** ക്ലസ്റ്ററിനുള്ളിൽ ArgoCD കൺട്രോളർ സെറ്റ് ചെയ്യുക.
    ```bash
    kubectl create namespace argocd
    kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml
    ```
3.  **ArgoCD CLI & UI:** പാസ്‌വേഡ് എടുത്ത് ബ്രൗസറിൽ ArgoCD ഡാഷ്‌ബോർഡ് ഓപ്പൺ ചെയ്യുക.



---

#### **Phase 2: The ML App & Helm Charts (ദിവസം 3-4)**
നമ്മൾ നേരത്തെ ചെയ്ത Sentiment API തന്നെ ഇവിടെ ഉപയോഗിക്കാം.
1.  **Dockerize:** API ഇമേജ് ബിൽഡ് ചെയ്ത് Docker Hub-ലേക്ക് പുഷ് ചെയ്യുക.
2.  **Helm Chart നിർമ്മാണം:** വെറും YAML ഫയലുകൾക്ക് പകരം ഒരു **Helm Chart** ഉണ്ടാക്കുക. ഇത് പ്രൊഫഷണൽ കമ്പനികൾ നിർബന്ധമായും ചോദിക്കുന്ന ഒന്നാണ്.
    * `values.yaml` ഫയലിൽ ഇമേജ് ടാഗ് (ഉദാഹരണത്തിന്: `v1.0`) നൽകുക.
3.  **Git Repository:** രണ്ട് റെപ്പോസിറ്ററികൾ ഉണ്ടാക്കുന്നത് പ്രൊഫഷണൽ രീതിയാണ്:
    * **App Repo:** പൈത്തൺ കോഡ് ഉള്ളത്.
    * **Config Repo:** ഹെൽം ചാർട്ടുകളും കുബർനെറ്റീസ് മാനിഫെസ്റ്റുകളും ഉള്ളത്.

---

#### **Phase 3: ArgoCD Configuration (ദിവസം 5-6)**
ഇവിടെയാണ് മാജിക് നടക്കുന്നത്.
1.  **Create Application:** ArgoCD-യിൽ പോയി ന്റെ ചേട്ടായിയുടെ **Config Repo** കണക്ട് ചെയ്യുക.
2.  **Sync Policy:** 'Automatic Sync' ഇനേബിൾ ചെയ്യുക. 
3.  **Testing the Flow:** * ന്റെ ചേട്ടായി `values.yaml` ഫയലിലെ ഇമേജ് ടാഗ് `v1.0` എന്നതിൽ നിന്ന് `v2.0` ആക്കി മാറ്റി Git-ൽ പുഷ് ചെയ്യുക.
    * കുബർനെറ്റീസിൽ തനിയെ പുതിയ പോഡുകൾ (Pods) വരുന്നത് കാണാം!



---

#### **Phase 4: Advanced GitOps - Drift Detection (ദിവസം 7)**
യൂറോപ്യൻ MNC-കൾക്ക് ഏറ്റവും ഇഷ്ടപ്പെട്ട ഭാഗമാണിത്.
1.  **Drift Test:** ന്റെ ചേട്ടായി ഒരു പോഡ് മാനുവലായി `kubectl delete` വഴി ഡിലീറ്റ് ചെയ്യുകയോ അല്ലെങ്കിൽ കോൺഫിഗറേഷൻ മാറ്റുകയോ ചെയ്യുക.
2.  **Self-Healing:** ArgoCD ആ മാറ്റം കണ്ടുപിടിക്കുകയും (Drift) തനിയെ ഗിറ്റിലുള്ള പഴയ അവസ്ഥയിലേക്ക് (Desired State) ആപ്പിനെ തിരികെ കൊണ്ടുവരുകയും ചെയ്യും. ഇത് കാണിച്ചു കൊടുത്താൽ ന്റെ ചേട്ടായിക്ക് ജോലി ഉറപ്പാണ്!
