## 🛠️ End-to-End Roadmap

### **Phase 1: Cluster & ArgoCD Setup (Day 1–2)**
- **Kubernetes Cluster:** Minikube/K3s (local) അല്ലെങ്കിൽ AWS Free Tier EC2.  
- **ArgoCD Installation:** ArgoCD controller install ചെയ്ത് UI + CLI configure ചെയ്യുക.  
- **Outcome:** ArgoCD dashboard accessible, cluster ready.  

---

### **Phase 2: ML App + Helm Charts (Day 3–4)**
- **ML API:** Use your existing Sentiment API project.  
- **Dockerization:** Build Docker image, push to Docker Hub.  
- **Helm Charts:** Create Helm chart with `values.yaml` (image tag, replicas, etc.).  
- **Repos:**  
  - **App Repo:** Python code.  
  - **Config Repo:** Helm charts + Kubernetes manifests.  
- **Outcome:** Professional repo separation (industry standard).  

---

### **Phase 3: ArgoCD Application & Sync (Day 5–6)**
- **Connect Config Repo:** ArgoCD application points to Config repo.  
- **Enable Auto Sync:** Git → Cluster sync enabled.  
- **Test Flow:** Change image tag in `values.yaml` → push to Git → ArgoCD auto deploys new pods.  
- **Outcome:** Recruiter‑ക്ക് കാണിക്കാൻ “No kubectl apply, Git change → auto deploy” demo.  

---

### **Phase 4: Drift Detection & Self-Healing (Day 7)**
- **Manual Change:** Delete pod or edit config via `kubectl`.  
- **ArgoCD Drift Detection:** ArgoCD detects drift, reverts cluster back to Git desired state.  
- **Outcome:** Recruiter‑ക്ക് കാണിക്കാൻ “Self-healing GitOps system” demo.
