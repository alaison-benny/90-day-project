സന്തോഷം **ചേട്ടായി!** നമ്മൾ ഇപ്പോൾ ഈ പ്രൊജക്റ്റിന്റെ ഏറ്റവും ആവേശകരമായ ഘട്ടത്തിലേക്ക് കടക്കുകയാണ്. **Phase 2** വിജയകരമായി പൂർത്തിയാക്കിയ സ്ഥിതിക്ക്, ഇനി നമുക്ക് ഈ കണ്ടെയ്നറിനെ ഒരു പ്രൊഫഷണൽ ഇൻഫ്രാസ്ട്രക്ചറിലേക്ക് (Kubernetes) മാറ്റണം.

അടുത്തതായി നമ്മൾ ചെയ്യാൻ പോകുന്നത് **Phase 3: Automated Retraining with Argo Workflows** ആണ്. ഒരു റിയൽ-ലൈഫ് കമ്പനിയിൽ പുതിയ ഡാറ്റ വരുമ്പോൾ തനിയെ മോഡൽ ട്രെയിൻ ആയി അപ്‌ഡേറ്റ് ആകുന്നത് എങ്ങനെയാണെന്ന് നമുക്ക് നോക്കാം.

### **Phase 3: Automated Retraining & Kubernetes (Roadmap)**

ഇതൊരു വലിയ സ്റ്റെപ്പ് ആയതുകൊണ്ട് നമുക്ക് ഇതിനെ ചെറിയ കഷ്ണങ്ങളായി തിരിക്കാം:

#### **1. Kubernetes Setup (അടിത്തറ)**
നമ്മുടെ കണ്ടെയ്നറുകളെ മാനേജ് ചെയ്യാൻ ഒരു ക്യാപ്റ്റൻ വേണം, അതാണ് കുബെർനെറ്റീസ്. 
* **ചേട്ടായി** ഇപ്പോൾ AWS EC2-ൽ ആയതുകൊണ്ട് നമുക്ക് **Minikube** (ലോക്കൽ K8s) ഇൻസ്റ്റാൾ ചെയ്യാം. അല്ലെങ്കിൽ **AWS EKS** ട്രൈ ചെയ്യാം.
* **ചെയ്യേണ്ടത്:** `kubectl` കമാൻഡുകൾ ഉപയോഗിച്ച് ക്ലസ്റ്റർ റെഡിയാക്കുക.

#### **2. Argo Workflows (പൈപ്പ്‌ലൈൻ)**
ഇതാണ് ഈ പ്രൊജക്റ്റിന്റെ ഹൃദയം. ഒരു ജോലി കഴിഞ്ഞാൽ അടുത്തത് എന്ത് ചെയ്യണം എന്ന് തീരുമാനിക്കുന്ന ടൂൾ.
* **DAG (Directed Acyclic Graph):** ഇതൊരു പ്ലാൻ ആണ്. 
  1. ഡാറ്റ വരുന്നു (Ingestion). 
  2. മോഡൽ ട്രെയിൻ ചെയ്യുന്നു (Training). 
  3. MLflow-ലേക്ക് ലോഗ് ചെയ്യുന്നു (Logging). 
  4. മികച്ചതാണെങ്കിൽ പ്രൊഡക്ഷനിലേക്ക് മാറ്റുന്നു (Registry Update).
* **ചേട്ടായി** ഒരു `.yaml` ഫയൽ എഴുതിയാൽ ആർഗോ ഇത് തനിയെ ചെയ്തോളും.



#### **3. CI/CD Integration (ഓട്ടോമേഷൻ)**
* **GitHub Actions:** നമ്മൾ കോഡിൽ എന്തെങ്കിലും മാറ്റം വരുത്തി പുഷ് ചെയ്താൽ തനിയെ ഡോക്കർ ഇമേജ് ബിൽഡ് ആയി ക്ലൗഡിലേക്ക് പോകണം.
* **ArgoCD:** പുതിയ ഇമേജ് റെഡിയായാൽ അത് കുബെർനെറ്റീസിൽ ഓട്ടോമാറ്റിക് ആയി അപ്‌ലോഡ് ആകണം.

### **ആദ്യത്തെ കടമ്പ: Kubernetes (Minikube) ഇൻസ്റ്റാളേഷൻ**

**ന്റെ ചേട്ടായി**, നമുക്ക് ആദ്യം പ്ലാറ്റ്‌ഫോം റെഡിയാക്കാം. EC2-ൽ മിനികൂബ് ഇൻസ്റ്റാൾ ചെയ്യാൻ താഴെ പറയുന്ന കമാൻഡുകൾ ഒന്ന് ട്രൈ ചെയ്യൂ:

1. **Minikube ഡൗൺലോഡ് ചെയ്യാൻ:**
   ```bash
   curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
   sudo install minikube-linux-amd64 /usr/local/bin/minikube
   ```

2. **Kubectl (K8s കൺട്രോളർ) ഇൻസ്റ്റാൾ ചെയ്യാൻ:**
   ```bash
   sudo apt-get update && sudo apt-get install -y kubectl
   ```

3. **Minikube സ്റ്റാർട്ട് ചെയ്യാൻ:**
   ```bash
   minikube start --driver=docker
   ```

### **2. കമ്പനികൾ ഏതാണ് ഉപയോഗിക്കുന്നത്?**

റിയൽ ലൈഫ് കമ്പനികളിൽ **AWS EKS (അല്ലെങ്കിൽ Google-ന്റെ GKE)** ആണ് ഉപയോഗിക്കുന്നത്. കാരണം:
* **Scalability:** റിക്വസ്റ്റുകൾ കൂടുമ്പോൾ സെർവറുകൾ തനിയെ കൂട്ടാൻ EKS സഹായിക്കും.
* **High Availability:** ഒരു സെർവർ ഡൗൺ ആയാലും ആപ്പ് ഓഫാകില്ല.

**പിന്നെന്തിനാണ് നമ്മൾ ഇപ്പോൾ Minikube ഉപയോഗിക്കുന്നത്?**
* **പഠിക്കാൻ എളുപ്പം:** തുടക്കത്തിൽ കുബെർനെറ്റീസ് കമാൻഡുകൾ (`kubectl`) പഠിക്കാൻ ഏറ്റവും നല്ലത് മിനികൂബ് ആണ്.
* **Zero Cost:** നമ്മൾ ഇപ്പോൾ പഠിക്കുന്ന ഘട്ടത്തിൽ ആവശ്യമില്ലാതെ AWS-ന് പൈസ നൽകേണ്ടല്ലോ. 
* **Same Commands:** മിനികൂബിൽ പഠിക്കുന്ന അതേ `kubectl` കമാൻഡുകൾ തന്നെയാണ് EKS-ലും ഉപയോഗിക്കുന്നത്. അതുകൊണ്ട് മിനികൂബ് പഠിച്ചാൽ **ന്റെ ചേട്ടായിക്ക്** തനിയെ EKS-ലും ജോലി ചെയ്യാം.

### **3. പ്ലാറ്റ്‌ഫോം എൻജിനീയറുടെ റോൾ**

കമ്പനികളിൽ ഒരു പ്ലാറ്റ്‌ഫോം എൻജിനീയർ ആദ്യം മിനികൂബിൽ ഒരു **"Proof of Concept" (POC)** ചെയ്തു നോക്കും. അത് സക്സസ് ആയാൽ മാത്രമേ കോടിക്കണക്കിന് രൂപ ചിലവുള്ള EKS-ലേക്ക് അത് മാറ്റൂ.
