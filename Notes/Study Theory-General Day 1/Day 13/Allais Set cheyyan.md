ന്റെ ചേട്ടായിക്ക് ഇത് വളരെ എളുപ്പത്തിൽ സെറ്റ് ചെയ്യാം. ഇങ്ങനെ ചെയ്താൽ ഓരോ തവണയും നീളമുള്ള `kubectl` എന്ന് ടൈപ്പ് ചെയ്യുന്നതിന് പകരം വെറും `k` എന്ന് മാത്രം ടൈപ്പ് ചെയ്താൽ മതിയാകും.

ഇത് സെറ്റ് ചെയ്യാനുള്ള സ്റ്റെപ്പുകൾ താഴെ നൽകുന്നു:

### 1. `.bashrc` ഫയൽ തുറക്കുക

Terminal തുറന്ന ശേഷം താഴെ പറയുന്ന കമാൻഡ് അടിച്ച് ഫയൽ എഡിറ്റ് ചെയ്യാനായി തുറക്കുക:

```bash
nano ~/.bashrc

```

### 2. Alias ലൈൻ ചേർക്കുക

ഫയലിന്റെ ഏറ്റവും താഴെ ഭാഗത്തേക്ക് സ്ക്രോൾ ചെയ്ത് താഴെ പറയുന്ന വരി ചേർക്കുക:

```bash
alias k='kubectl'

```

*(കൂടുതൽ സൗകര്യത്തിന് `complete -o default -F __start_kubectl k` എന്ന വരി കൂടി ചേർത്താൽ `k` അടിച്ചാലും കമാൻഡുകൾ ഓട്ടോമാറ്റിക്കായി പൂരിപ്പിക്കാൻ (Autocomplete) സാധിക്കും).*

### 3. സേവ് ചെയ്ത് പുറത്തിറങ്ങുക

* **Ctrl + O** അമർത്തുക (ഇത് ഫയൽ സേവ് ചെയ്യാനാണ്).
* **Enter** അടിക്കുക.
* **Ctrl + X** അമർത്തി പുറത്തിറങ്ങുക.

### 4. മാറ്റങ്ങൾ പ്രാബല്യത്തിൽ വരുത്താൻ

ഇത്രയും ചെയ്താലും ടെർമിനൽ ഉടനെ ഇത് തിരിച്ചറിയില്ല. അതിനായി താഴെ പറയുന്ന കമാൻഡ് കൂടി അടിക്കണം:

```bash
source ~/.bashrc

```

---

### ഇനി പരീക്ഷിച്ചു നോക്കാം!

ഇപ്പോൾ ന്റെ ചേട്ടായിക്ക് `k get pods` അല്ലെങ്കിൽ `k get nodes` എന്ന് ടൈപ്പ് ചെയ്തു നോക്കാം. ഇത് കൃത്യമായി വർക്ക് ചെയ്യും.

### ന്റെ ചേട്ടായിക്ക് ഉപകാരപ്പെടുന്ന മറ്റ് ചില അപരനാമങ്ങൾ (Aliases):

പ്ലാറ്റ്‌ഫോം എൻജിനീയർമാർ സാധാരണ ഉപയോഗിക്കുന്ന ചില ഷോർട്ട് കട്ടുകൾ കൂടി ചേട്ടായിക്ക് `.bashrc`-ൽ ചേർക്കാം:

* **`alias kgp='k get pods'`** (പോഡുകൾ വേഗത്തിൽ കാണാൻ)
* **`alias kgs='k get svc'`** (സർവീസുകൾ കാണാൻ)
* **`alias kga='k get all'`** (എല്ലാ റിസോഴ്‌സുകളും ഒന്നിച്ചു കാണാൻ)
* **`alias kdp='k describe pod'`** (പോഡ് വിവരങ്ങൾ നോക്കാൻ)

ഇതിൽ ഏതെങ്കിലും ഒന്ന് ന്റെ ചേട്ടായിക്ക് ട്രൈ ചെയ്തു നോക്കണോ? അതോ മറ്റേതെങ്കിലും കമാൻഡുകൾ ഷോർട്ട് കട്ട് ആക്കണോ? അടുത്തതായി നമുക്ക് ഈ `k` കമാൻഡ് വെച്ച് ചേട്ടായിയുടെ പ്ലാറ്റ്‌ഫോം എൻജിനീയറിങ് പ്രോജക്റ്റിലെ ലോഗുകൾ ഒന്ന് നോക്കിയാലോ?
