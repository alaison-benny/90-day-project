ന്റെ ചേട്ടായിക്ക് പ്ലാറ്റ്‌ഫോം എൻജിനീയറിങ്ങിൽ ഏറ്റവും കൂടുതൽ സമയം ചിലവഴിക്കേണ്ടി വരുന്നത് ഈ **kubectl** കമാൻഡുകൾക്കൊപ്പമായിരിക്കും. Kubernetes ക്ലസ്റ്ററിനോട് സംസാരിക്കാനുള്ള നമ്മുടെ പ്രധാന ആയുധമാണിത്.

ഇത് എങ്ങനെ ഇൻസ്റ്റാൾ ചെയ്യാമെന്നും നിത്യേന ഉപയോഗിക്കേണ്ട പ്രധാന കമാൻഡുകൾ ഏതൊക്കെയാണെന്നും താഴെ നൽകുന്നു.

---

### 1. kubectl ഇൻസ്റ്റാൾ ചെയ്യുന്ന വിധം (Installation)

ന്റെ ചേട്ടായി ഉപയോഗിക്കുന്ന ഓപ്പറേറ്റിംഗ് സിസ്റ്റം അനുസരിച്ച് ഇൻസ്റ്റലേഷൻ മാറാം. ചേട്ടായി Ubuntu ആണ് ഉപയോഗിക്കുന്നത് എന്ന് എനിക്കറിയാം, അതുകൊണ്ട് Linux-ലെ രീതി താഴെ നൽകുന്നു:

**Linux (Ubuntu) വഴി:**

```bash
# 1. ലേറ്റസ്റ്റ് ബൈനറി ഡൗൺലോഡ് ചെയ്യാൻ
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"

# 2. പെർമിഷൻ മാറ്റാൻ
chmod +x ./kubectl

# 3. സിസ്റ്റം പാത്തിലേക്ക് മാറ്റാൻ
sudo mv ./kubectl /usr/local/bin/kubectl

# 4. ഇൻസ്റ്റലേഷൻ ഉറപ്പുവരുത്താൻ
kubectl version --client

```

### 2. എങ്ങനെ അപ്‌ഗ്രേഡ് ചെയ്യാം? (Upgrade)

kubectl അപ്‌ഗ്രേഡ് ചെയ്യാൻ പ്രത്യേക കമാൻഡ് ഇല്ല. മുകളിൽ പറഞ്ഞ അതേ സ്റ്റെപ്പുകൾ വീണ്ടും ചെയ്താൽ മതി. പുതിയ വേർഷൻ പഴയ ഫയലിനെ ഓവർറൈറ്റ് (Overwrite) ചെയ്തോളും.

### 3. നിത്യേന ഉപയോഗിക്കേണ്ട പ്രധാന കമാൻഡുകൾ (Daily Use)

ന്റെ ചേട്ടായിക്ക് ജോലി എളുപ്പമാക്കാൻ ഈ കമാൻഡുകൾ താഴെ പറയുന്ന കാറ്റഗറികളായി തിരിക്കാം:

#### **A. കാര്യങ്ങൾ പരിശോധിക്കാൻ (Get & Describe)**

* `kubectl get nodes`: ക്ലസ്റ്ററിലെ സെർവറുകളുടെ ലിസ്റ്റ് കാണാൻ.
* `kubectl get pods`: നിലവിൽ റൺ ചെയ്യുന്ന ആപ്പുകൾ കാണാൻ.
* `kubectl get svc`: സർവീസുകൾ (IP/Port) കാണാൻ.
* `kubectl describe pod [pod-name]`: ഒരു പോഡിന് എന്തെങ്കിലും പ്രശ്നമുണ്ടോ എന്ന് വിശദമായി പരിശോധിക്കാൻ.

#### **B. പ്രശ്നങ്ങൾ കണ്ടെത്താൻ (Troubleshooting)**

* `kubectl logs [pod-name]`: ആപ്പിന്റെ ഉള്ളിലെ എററുകൾ (Logs) കാണാൻ.
* `kubectl logs -f [pod-name]`: ലൈവ് ആയി ലോഗ്സ് കാണാൻ.
* `kubectl exec -it [pod-name] -- bin/bash`: ഒരു പോഡിന്റെ ഉള്ളിൽ കയറി കമാൻഡുകൾ അടിക്കാൻ.

#### **C. കോൺഫിഗറേഷൻ മാറ്റാൻ (Apply & Delete)**

* `kubectl apply -f [file.yaml]`: ഒരു പുതിയ ഫയൽ ഡിപ്ലോയ് ചെയ്യാൻ.
* `kubectl delete -f [file.yaml]`: ഡിപ്ലോയ് ചെയ്ത ആപ്പ് ഒഴിവാക്കാൻ.
* `kubectl edit deployment [name]`: റൺ ചെയ്യുന്ന ഒരു ഡിപ്ലോയ്‌മെന്റിൽ ലൈവ് ആയി മാറ്റം വരുത്താൻ.

#### **D. പശ്ചാത്തല വിവരങ്ങൾ (Context & Namespace)**

* `kubectl config get-contexts`: ഏത് ക്ലസ്റ്ററിലാണ് ഇപ്പോൾ ഉള്ളതെന്ന് നോക്കാൻ.
* `kubectl get ns`: എല്ലാ നെയിംസ്‌പേസുകളും കാണാൻ.
* `kubectl get pods -n [namespace-name]`: ഒരു പ്രത്യേക നെയിംസ്‌പേസിലെ പോഡുകൾ കാണാൻ.

### ന്റെ ചേട്ടായിക്ക് ഒരു പ്ലാറ്റ്‌ഫോം എൻജിനീയറിങ് ടിപ്പ് (Aliases):

എപ്പോഴും `kubectl` എന്ന് ടൈപ്പ് ചെയ്യുന്നത് ബുദ്ധിമുട്ടാണെങ്കിൽ ചേട്ടായിക്ക് ഇത് ചുരുക്കി ഉപയോഗിക്കാം.
`alias k='kubectl'` എന്ന് ചേട്ടായിയുടെ `.bashrc` ഫയലിൽ കൊടുത്താൽ മതി. പിന്നീട് `k get pods` എന്ന് അടിച്ചാൽ മതിയാകും.
